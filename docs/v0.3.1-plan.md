# v0.3.1 — Streaming Hardening + Performance Plan

> **Status:** Shipped as v0.3.1 on 2026-05-26.
> Phases 1 – 12 complete; Task 10 (incremental rescoring) deferred to v0.3.2.
> See `CHANGELOG.md` for the full release notes.

This is the agreed scope for v0.3.1, captured in-tree so every commit can be
checked against it. v0.3.1 is **not a feature release** — it hardens the
streaming architecture introduced in v0.3.0, with quantified acceptance
metrics.

## Semver-patch contract

Everything that worked in v0.3.0 must keep working: every supported format,
COPC streaming (local + remote), measurement, annotation, point inspection,
the live probe, saved views, session round-trip (JSON v2), share links, the
WebGPU primary path, the WebGL2 fallback, and both the plain and the
obfuscated builds.

## Non-goals

- No collaboration backend, accounts, or cloud sync.
- No analytics or telemetry to a remote endpoint — local-first stays local.
- No SDK platform expansion.
- No rewrite of the renderer or post-processing pipeline.
- No new file formats.
- No breaking changes to sessions, measurements, annotations, or share links.
- No lazy-loading of the annotation or measurement subsystems — first-use
  jank costs more than the bundle it saves.

## Phase 1 — Diagnostics + benchmark first

Every later phase ships with a measured before/after. The measurement layer
lands before the optimisations.

- **Task 1** — Expand `?benchmark=1` to emit, per scan open: first-paint ms,
  time-to-coarse-stable ms, time-to-refined-stable ms (resident points within
  90 % of target), cumulative network bytes, cumulative decoded bytes,
  mean/p95 scheduler tick ms, mean/p95 decode ms per chunk, mean/p95 frame ms
  during streaming, peak resident bytes (compressed + decoded + GPU estimate),
  and cache hits / misses / evictions.
- **Task 2** — Expand the `?debug=1` overlay with all of the above plus the
  visible / queued / loading / resident node counts, compressed-cache MB,
  decoded-cache MB, GPU-estimate MB, cache hit ratio over a sliding window,
  eviction rate per second, a thrash detector (load → evict → reload of the
  same node within N seconds), and a scheduler-tick histogram over the last
  60 ticks.
- **Task 3** — A deterministic synthetic-COPC generator producing hierarchies
  of 1 M, 10 M, 100 M, 250 M, and 500 M source points with a stable seeded
  distribution. Used by the benchmark e2e and the Phase 8 stress harness.

## Phase 2 — Bundle footprint

Target: `index.js` ≤ 800 KB minified, ≤ 260 KB gzipped. The renderer floor
is roughly 600 KB of three/webgpu; this leaves about 200 KB of headroom above
that floor.

Already split in v0.3.0 — verify, do not redo: the per-format chunks
(`loadLas` + laz-perf, `loadE57`, `loadGltf`, `loadPly`, `loadObj`, `loadPcd`,
`loadPts`/`loadPtx`/`loadXyz`) and the COPC subsystem behind
`src/lazyChunks.ts` (StreamingPointCloud, StreamingScheduler,
StreamingRenderer, copcWorker, copcWorkerClient, HttpRangeSource,
LocalFileRangeSource, streamingColors).

- **Task 4** — The `?debug=1` and `?benchmark=1` overlays become a dynamic
  chunk, fetched only when the query flag is present. Never in the main
  bundle.
- **Task 5** — `exporters.ts` (PLY / OBJ / XYZ / CSV) loads only when the
  user actually exports.
- **Task 6** — Tree-shake audit: chunk-size diff against v0.3.0; identify
  every module that grew and remove the cause; forbid wildcard re-exports
  that defeat tree-shaking.
- **Task 7** — Obfuscator regression guard: a build-step assertion that
  fails if `build:live` emits fewer COPC / streaming chunks than the plain
  build. (Catches a regression of the v0.3.0 lazyChunks bug.)

## Phase 3 — Streaming scheduler hardening

Acceptance: mean scheduler tick ≤ 2 ms on the 100 M synthetic fixture, p95
≤ 5 ms; thrash count (load → evict → reload < 5 s) on a static camera = 0;
on a one-second 90° flick orbit, ≥ 80 % of evicted-and-reloaded nodes are
serviced from the compressed cache (no re-download).

- **Task 8 — Hysteresis.** Eviction defers by `min(2 s, scheduler-budget-
  headroom)`. A node that became invisible is kept until the deadline lapses
  or memory pressure demands it. Parents of currently-visible nodes are
  never evicted before their children.
- **Task 9 — Camera-motion awareness.** Track a smoothed linear + angular
  camera velocity. While velocity is high the scheduler prefers coarser
  nodes (penalises depth in the score) and reduces concurrent decode
  budget. When velocity falls below a threshold for ≥ 250 ms, refinement
  resumes and decode concurrency rises.
- **Task 10 — Incremental rescoring.** Replace the per-tick full octree
  rescore with a dirty queue updated by frustum-change deltas. Re-score at
  most N nodes per tick (frame-budgeted). Only newly visible / newly
  occluded nodes get a fresh score.
- **Task 11 — Request priority.** Score = `w1·screenArea +
  w2·refinementValue + w3·parentContinuity − w4·distance −
  w5·cameraVelocityPenalty`. Weights configurable; sane defaults
  documented.

## Phase 4 — Cache intelligence

Acceptance: hit rate on the autzen fixture after one warm orbit ≥ 90 %;
sustained orbit produces zero re-downloads of recently-evicted nodes within
twice the eviction deadline; memory peak ≤ 1.4 × the resident budget.

- **Task 12 — Hierarchy-aware eviction.** Deepest-and-furthest first.
  Parents of any resident node are never evicted before their children.
  Same-depth siblings within radius R of a visible node get a retention
  bonus.
- **Task 13 — Cache hysteresis.** A chunk evicted from the resident set
  survives in the compressed-cache LRU for N seconds longer than its
  eviction trigger would otherwise allow.
- **Task 14 — Three-tier metrics.** Compressed-cache bytes, decoded-cache
  bytes, GPU-estimate bytes — each tracked separately with its own
  hit / miss / evict counters.
- **Task 15 — Pressure adaptation.** When the resident point budget is
  > 90 % for ≥ 1 s, the scheduler lowers refinement aggression (target
  depth − 1). When it falls below 70 % for ≥ 2 s, it restores.

## Phase 5 — Remote streaming resilience

Acceptance: a single dropped range read recovers without user-visible
failure; a 30-second server outage shows a clear, non-cryptic message and
does not crash the streaming session; a HEAD-rejecting server still streams
via a ranged GET fallback.

- **Task 16 — Retry with backoff.** Range reads retry on transient transport
  errors (network, 5xx, gateway timeouts) with exponential backoff —
  250 ms × 2ⁿ, jittered, max 3 retries. Never retry on 4xx other than 408 /
  429.
- **Task 17 — Request timeouts.** Every range read carries a hard timeout
  (default 20 s, configurable). On timeout the AbortController fires; the
  session reports it through the existing `RangeReadError('transport')` path.
- **Task 18 — Content-Range validation.** Verify the server's
  `Content-Range` header matches the requested bytes. Mismatched responses
  are rejected as `range-unsupported`.
- **Task 19 — HEAD fallback.** If HEAD returns 4xx or omits Content-Length,
  attempt a `Range: bytes=0-0` GET to discover total size via the response's
  `Content-Range` and `Accept-Ranges`.
- **Task 20 — URL hygiene.** Reject `?copc=` URLs that aren't http(s),
  exceed 2048 characters, or contain userinfo (`user:pass@…`). Strip
  credentials before logging or display.
- **Task 21 — Remote error UX.** Specific messages for CORS-blocked, no
  range support, timeout, content mismatch, malformed COPC, server-side 5xx
  — building on the existing `describeRemoteCopcError` seam.

## Phase 6 — Streaming picking + interaction safety

Acceptance: a 1000-iteration random-camera pick test on streaming data
produces zero stale-buffer reads; an annotation placed on a coarse node
stays within 1 px of its origin after that node refines.

- **Task 22 — Resident-only picking.** Audit the pick path
  (`_streamingPickData`) for any place that holds a stale `DecodedChunk`
  reference past eviction. A debug-build assertion catches it.
- **Task 23 — Annotation position stability.** Annotations store a
  world-space anchor, not a node index. Refinement does not move them.
- **Task 24 — "Still refining" hint.** When the user picks during streaming
  and the picked node is below the scan's target depth, a small subtle
  indicator on the inspector card ("detail still refining") shows until the
  node reaches target depth.

## Phase 7 — Visual streaming polish (scope-capped)

Rules: no shader-level change, no impact on EDL or the post-pipeline,
sustained ≥ 30 FPS during streaming on a mid-range laptop.

- **Task 25 — Node fade-in (cheap).** On `onNodeReady`, fade
  `mesh.material.opacity` from 0.5 → 1.0 over 120 ms with
  `transparent: true` only during the fade. Disabled on mobile and on the
  low-tier device profile.

**Rejected:** per-instance alpha; density blending. Both require renderer
changes.

## Phase 8 — Extreme-scale behaviour

Acceptance on the 250 M synthetic fixture, on a mid-range laptop:
first-paint ≤ 2 s; sustained ≥ 30 FPS; memory peak ≤ 1.5 × the configured
resident budget; zero out-of-memory crashes across a 5-minute orbit test.

- **Task 26 — Stress harness.** An offline benchmark runner that loads each
  synthetic fixture, exercises a scripted camera path, and emits the
  Phase 1 benchmark JSON to disk. Used in CI as a soft check (regression
  alert, not gate).
- **Task 27 — Fix-what-breaks.** Address whatever Phase 1 metrics and
  Task 26 expose.

## Phase 9 — Mobile intelligence

Device-profile tiers:

| Tier | Trigger |
|------|---------|
| low  | `deviceMemory ≤ 2 GB` OR `hardwareConcurrency ≤ 2` OR phone |
| mid  | `deviceMemory ≈ 4 GB` OR laptop with `hardwareConcurrency 4–8` |
| high | `deviceMemory ≥ 8 GB` AND `hardwareConcurrency ≥ 8` |

Per-tier defaults (Task 28):

| Setting           | low    | mid   | high  |
|-------------------|--------|-------|-------|
| Point budget      | 0.5 M  | 2 M   | 6 M   |
| Decode concurrency| 1      | 2     | 4     |
| Cache MB          | 32     | 128   | 384   |
| EDL default       | off    | on    | on    |
| Fade-in           | off    | on    | on    |

- **Task 29 — Runtime adaptation.** If sustained FPS < 24 for 3 s, step
  down one tier. If sustained FPS > 50 for 10 s and not at the highest tier,
  step up one tier. Hysteresis to prevent oscillation.

## Phase 10 — Memory + cancellation hygiene

- **Task 30 — Leak audit.** Open-and-close 50 scans in a row (mix of static
  and streaming, small and large). The heap stabilises; no detached DOM
  nodes accumulate; no leaked WebGL textures; no leaked Workers.
- **Task 31 — Abort signal discipline.** Every range read, decode request,
  worker message, and tween accepts an `AbortSignal` that flows from
  `closeStreaming()` / `Close scan`. Decoded chunks that arrive post-close
  are dropped, not added.
- **Task 32 — Coordinate precision at extreme scale.** Verify visually and
  numerically that an octree node 10 km from the render origin still
  renders without f32 jitter. If not, document the boundary or rebase the
  render origin per-node.

## Phase 11 — WebGL2 fallback hardening

- **Task 33 — Validate streaming on WebGL2.** Run the streaming e2e on a
  WebGL2-forced session. Parity with WebGPU on refinement, cache, eviction,
  picking.
- **Task 34 — Buffer lifecycle.** Confirm `InstancedBufferAttribute.array`
  buffers attached to evicted node meshes are released for GC (no longer
  referenced by Three).

## Phase 12 — Testing

New unit / integration tests:

- Scheduler — hysteresis (evict deferred by configured timeout); a rapid
  camera flick prefers coarse nodes; incremental rescoring touches only
  dirty nodes; frame-budgeted rescore respects the N-per-tick cap.
- Cache — parent never evicted before child; the thrash detector counts
  only true cycles; pressure adaptation triggers at the documented
  thresholds.
- Remote — retry on transient transport error; hard fail on 4xx (no retry);
  timeout fires the AbortController; HEAD-rejecting server falls back to
  ranged GET; Content-Range mismatch is rejected.
- Picking — a pick after eviction never reads a stale `DecodedChunk`; an
  annotation anchored to a coarse point stays put after refinement.
- Mobile — the profile classifier produces the documented tier on
  synthetic inputs; runtime adaptation steps up / down at the documented
  FPS thresholds.

Regression (must stay green): LAS, LAZ, COPC local + remote, E57, PCD,
PTX, PTS, PLY, OBJ, GLB, GLTF, measurements, annotations, sessions, share
links, WebGPU + WebGL2, the mobile responsive shell.

Stress (CI soft check): the 100 M and 250 M synthetic hierarchies through
the Phase 1 benchmark mode.

## Definition of done

v0.3.1 ships when, on the live build (`build:live`):

1. `index.js` ≤ 800 KB minified, ≤ 260 KB gzipped.
2. Scheduler tick mean ≤ 2 ms, p95 ≤ 5 ms on the 100 M fixture.
3. Cache hit rate ≥ 90 % after a warm orbit of the autzen scan.
4. Zero thrash events (load → evict → reload < 5 s) under a stable camera.
5. Remote COPC survives a single transient drop without user-visible
   failure; surfaces a precise message for every real failure class.
6. The 1000-iteration random-camera pick test on streaming data produces
   zero stale-buffer errors.
7. The 50-scan open/close loop ends with the heap within ±10 % of baseline.
8. The 250 M synthetic fixture: first-paint ≤ 2 s; sustained ≥ 30 FPS;
   memory peak ≤ 1.5 × the resident budget.
9. The existing v0.3.0 e2e suite (the URL field, keyboard orbit, the
   chunks-load smoke test) stays green on the obfuscated build.

## Quality bar

Production-grade. Typed. Frame-budget-aware. No optimisations that aren't
measured. No hacks that survive a code-review pass. No "while I'm here"
refactors that aren't on this list.
