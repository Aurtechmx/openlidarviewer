# v0.6.0-alpha.1 — response to the alpha engineering review

This records what changed in answer to the latest v0.6 alpha review (the one
that scored the build "strongest so far" and cleared it for *controlled
foundation development* but not *public-alpha publication*). Every item is
classified by how it was verified, because some of this work is provable in Node
and some is only observable in the browser — and the two must not be conflated.

The version is unchanged at `0.6.0-alpha.1`; the release manifest pins the exact
commit for traceability instead of bumping.

## Verification legend

- **Node-verified** — a failing-then-green unit test pins the behaviour; it runs
  in the release gate (`npm run test:release`, checked for the literal
  `GATE EXIT: 0`).
- **Browser-only** — the effect is visual/GPU and cannot be observed from Node;
  it needs a person watching the running app. Flagged, never claimed done.
- **Refuted** — investigated against the actual code and found not to apply to
  this build; no change made, evidence recorded.

## Reviewer blockers

### 1. No shared project coordinate frame — foundation landed, wiring staged

The largest architectural gap: each cloud mounts in its own `floor(min)` frame,
so two georeferenced scans with different origins both land near local zero and
appear overlaid. The **foundation** ships: `ProjectSpatialFrame` and
`LayerSpatialTransform` value types plus pure Float64 transform math — choose a
shared origin, map source-local ↔ project-local ↔ world, recover absolute
coordinates — with a unit test pinning the round-trip and the sub-mm residual
bound. **Node-verified.** The design and the staged, browser-verified scene
wiring (mounting every layer through the frame, Compare Studio, cross-layer
picking, persisting the frame in the schema) are written up in
[project-spatial-frame.md](../architecture/project-spatial-frame.md). The live
scene wiring is deliberately **not** in this build — it reshapes the project
schema and is verifiable only in the browser.

### 2. A session could rebase onto an unrelated scan — fixed

Session import rebased its measurements onto whatever cloud was loaded, silently
realigning one scan's analysis onto another. Import now checks the session's
stored scan fingerprint against the loaded scan first: extents are the primary
signal (they identify a scan spatially and survive device-driven downsampling),
so a spans mismatch beyond tolerance is refused; point count corroborates but
only downgrades to a disclosed partial match; file name and CRS label are
disclosure only. The matcher is pure and **Node-verified** (strong / partial /
conflict cases, including the rename and reduced-cloud edge cases).

### 3. Streaming nodes could emit non-finite coordinates — fixed

The central sanitiser cleans file-loaded clouds but by contract skips streaming
node buffers, and there was no finiteness check in the COPC/EPT decoders — a
malformed header scale/offset/origin, or a float NaN in an EPT tile, reached the
GPU untouched. The decoders now refuse such a node with a structured
malformed-file error that the scheduler's existing retry/back-off isolates. For
the integer COPC/EPT-laszip path an O(1) transform check is complete; the float
EPT-binary path also scans the finished positions. **Node-verified.**

### 4. PCD could reach a bounding computation with NaN — refuted

Investigated: the PCD loader already routes its parsed positions through
`sanitizeAndRecenter` before constructing the cloud, and there is no
`computeBoundingSphere` (or any three.js `BufferGeometry.boundingSphere`) call in
the codebase — the renderer draws instanced quads and `PointCloud.bounds()`
independently skips non-finite coordinates. The specific mechanism the review
described does not exist in this build. **Refuted**, no change made.

## Related correctness + tooling

### Density confidence without a known unit — fixed

The stockpile density confidence bar is a points/m² threshold, but an unknown
CRS still resolved to `linearUnitToMetres = 1`, so a native points/unit² density
was read as pts/m² and could award HIGH confidence. HIGH (the only tier that
rests on density) is now withheld when the unit is unknown, and the presenter
labels the density row honestly instead of claiming m². **Node-verified.**

### Reliable single-file test runner — added

Driving one test file red-to-green meant either a six-minute release bucket or an
ad-hoc vitest run that could print its summary and then hang, leaving no
trustworthy signal. A single-file runner mirrors the release buckets'
terminating policy and adds a wall-clock watchdog, so a hung shutdown becomes a
fast, explicit failure and the result is a greppable `TESTFILE EXIT` line rather
than an inferred one.

### Progressive EPT hierarchy attach — landed earlier this wave

Large public EPT datasets expose thousands of hierarchy sub-files; loading all
of them before first paint made big projects look hung. The walk is now split
into a bounded first-paint pass plus a resumable continuation, and the frontier
drops every attempted key so a persistently-failing fetch can't spin into an
allocation loop. **Node-verified**, including the fetch-failure termination
regression.

## Browser-only, honestly not closed here

### Streaming "regions pulsing" flicker

Root cause: at the point-budget boundary, a newcomer scoring marginally higher
after a camera nudge bumps an already-shown node out of the selection; that node
later lapses its evict-defer window, is dropped, and re-fades in when wanted
again. A resident-stickiness option was added to the pure budget selector and
**unit-tested** (holds the slot against a marginal newcomer, yields to a
decisively better one, exempts a node being refined so LOD never freezes). It is
left **opt-in and not wired into the live scheduler**: enabling it must first
reconcile the refining exemption with the scheduler's ancestor-protection (a
refining ancestor stripped of its bonus can go unwanted yet stay resident,
stalling eviction — observed in the eviction-contract tests) and then be
verified in the browser, where the flicker actually shows. Shipping it blind is
exactly the mistake an earlier admission-hysteresis attempt made.

## Verdict

**Ready for controlled v0.6 foundation development. Not yet ready for public-alpha
publication.** The concrete correctness defects the review named (2, 3, and the
density gate) are fixed and Node-verified; blocker 4 is refuted with evidence;
blocker 1's foundation is in place with the schema-reshaping wiring staged behind
it. The two items that remain are, by their nature, browser-verified — the
spatial-frame scene wiring and the flicker's live enablement — and are called out
here as staged against tested foundations, not folded blindly into the build.
