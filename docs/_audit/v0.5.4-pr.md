# PR: v0.5.4 — Terrain Science Hardening

Terrain complexity goes from a header-time heuristic to two literature-defined,
CI-guarded metrics with reproducible parameters and cited caveats. Branch:
`feat/v0.5.4-terrain-science`.

## Shipped

### Cores (pure-data leaves, `src/terrain/complexity/`)
- **TPI + six-class slope position** per Weiss (2001): shrinking windows,
  honest NoData, standardised TPI, tan(5°) flat/middle split on the existing
  rise/run slope; hand-computed analytic tests.
- **VRM** per Sappington et al. (2007), doi:10.2193/2005-723: unit normals
  decomposed from the EXISTING Horn slope/aspect grids (math-frame downslope
  aspect mapped to the Sappington decomposition — a relabelling that provably
  leaves R and VRM unchanged); slope-independence asserted analytically.
- **Honesty envelope**: every result carries `TerrainCoverageMeta` fields with
  a confidence DERIVED from valid-fraction × window support; median + IQR
  mandatory; ordered warnings.
- **Summary stage** (`complexitySummary.ts`): one compact, structured-clone-safe
  record per run — VRM median [IQR] + 3×3 window in cells AND ground metres;
  TPI dominant class + radius (targets ~10 m, clamped 2–10 cells, ACHIEVED
  radius stated) + Z-unit label; display band over the VRM median with the
  numbers always alongside; pre-formatted display strings so every surface
  prints the same words.

### Integration (Phase 3)
- Computed **inside `computeTerrainCore`** — the worker path (main-thread
  fallback included), cached with the core, never on the interactive path and
  never eagerly at attach. `TerrainCore` / `AnalyseContoursResult` gain
  `complexity: TerrainComplexitySummary | null`.
- **Dataset Intelligence**: the "Terrain Complexity" row is engine-fed via
  `inspectorCards.noteTerrainComplexity(...)` from the terrain runner's
  `onResult` (stale-guarded); the derived band overrides the heuristic only
  when a run measured; numeric median + IQR + window + units in the hover
  tooltip and Details row; "—" preserved when nothing is measured; the row's
  signal tier stays NEUTRAL (descriptive, never good/bad).
- **Analyse panel**: a derived-metrics line under the Terrain Assessment
  (`.olv-analyse-derived`), with the density caveat in the standard
  `.olv-caveat` treatment.
- **Reports/provenance**: `ExportProvenanceComplexity` — metric names,
  window/radius in cells AND ground units, Z units, Horn slope/aspect
  convention note, derived confidence, ordered caveats — stamped verbatim
  into README/DXF/SVG lines, GeoJSON metadata, and the terrain report's
  assessment rows + warnings. `provenanceConsistency.test.ts` extended to pin
  the new fields across every artifact.

### Density-reliability caveat (Phase 4, cited)
- Below 4 pts/m² (scan-scaled ground density, the existing stride-honest
  figure): "point density N pts/m² is below the ≥4 pts/m² reliability
  threshold reported for detailed terrain/vegetation complexity (Münzinger et
  al. 2022, doi:10.1016/j.ufug.2022.127637); treat complexity as indicative."
  Warning, never a block. Tests: present at 2 pts/m², absent at 6 (and at the
  threshold), none for unknown density. LaRue et al.
  (doi:10.5281/zenodo.6463393) cited in docs as the sensitivity evidence.

## Deferred (explicitly, with rationale in docs/terrain-intelligence.md)
- Multi-scale TPI with the ten-class landform scheme (Weiss 2001).
- Booth et al. (2009) wavelet/spectral curvature.
- Fractal dimension by variogram.
- Arc–chord ratio rugosity (Du Preez 2015) — the alternative slope-decoupled
  measure; VRM shipped first.
- PROTECT-style topographic cross-sections.
- Optional MCC ground filter (Evans & Hudak 2007) alongside SMRF.
- Per-segment lasso IDs for complexity-by-region (prior art: Papucci &
  Yrttimaa 2026).

## Honesty invariants preserved
- Confidence derived, never asserted; no floors, no constants.
- "—" wherever nothing was measured; bands never fabricated.
- Complexity remains a NEUTRAL (descriptive) signal — never coloured
  good/bad.
- Windows and units stated on every figure, in cells AND ground units.
- AGPL isolation: cores implemented from the primary literature only.
