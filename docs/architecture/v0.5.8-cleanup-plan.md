# v0.5.8 Architectural & Scientific Cleanup ‚Äî grounded plan

This turns the v0.5.8 cleanup master prompt (a 15-PR architecture program) into an
execution plan anchored to the **actual current code**, not aspiration. It was
written after a four-domain read-only audit of the `feat/v0.5.8-hardening` branch
(units, evidence/exports, validation, dependencies/bundle).

The master prompt is explicit: **do not merge this as one unreviewable refactor.**
This plan keeps that discipline ‚Äî each PR is independently reviewable, and the
sequence front-loads foundations that later PRs depend on.

## Hard constraints that shape sequencing

1. **Bundle is nearly full.** Any PR that adds eager weight to `index` must be
   offset. `main.ts` decomposition (PR13) is the lever; until then, new runtime
   code goes behind lazy boundaries or into workers.
2. **Method-registry / evidence-gate wiring touches export + UI paths** that are
   only fully verifiable on a real browser (e2e) and on device. Land pure cores
   first; stage UI/watermark wiring behind a device check.
3. **Do not fabricate validation.** E4+ still requires a real independent
   reference; nothing here manufactures it. Harnesses stay `pending`.

## Recommended sequence (maps to the master prompt's PR list)

Ordered by dependency and risk, not by the prompt's numbering.

### Tier 1 ‚Äî foundations others depend on (do first)
- **PR1 Build identity** (¬ß5). Generate one `BuildIdentity` at build time
  (commit, dirty, builtAt, node, channel) and feed UI/PDF/JSON/provenance.
  Removes hand-copied version strings (audit: `metricVersion` is free text).
  Small, additive; watch bundle.
- **PR4 Method registry** (¬ß8). `src/science/methodRegistry.ts` +
  `docs/science/METHOD_REGISTRY.md`. Give every algorithm a stable id@version
  (`olv.dtm.median-grid@2`, `olv.validation.spatial-block@2`, ‚Ä¶). Pure data;
  unblocks provenance and claim‚Üîmethod CI checks. **Absent today.**
- **PR2 Units wiring** (¬ß7). The typed system exists; wire it at the **one
  chokepoint** the audit found: `geo/CoordinateTypes.ts` `linearUnitToMetres`
  ‚Üí a `UnitScale` (known/unknown) with a separate vertical scale. This
  structurally kills the `pointInfo`/`spaceReportLayout` "metres (assumed)"
  leaks at the root. Add the ¬ß7.5 static check for `as Metres` in science.

### Tier 2 ‚Äî the scientific spine
- **PR3 Canonical analysis record** (¬ß6). One immutable, hashable,
  schema-versioned `ScientificAnalysisRecord` that every output derives from.
  Today four domains reconstruct provenance independently ‚Äî unify them.
- **PR5 Claim registry generation** (¬ß9). Move from the hand-authored runtime
  `EVIDENCE_REGISTRY` to a **build-time generated** `claimRegistry.generated.ts`
  from the YAML (no browser YAML parse), with CI failing on duplicate ids,
  productless claims, exporter-without-claim, prohibited wording.
- **PR6 Evidence gate everywhere** (¬ß10). One `resolveEvidenceDecision(record,
  claim)`; route **every** exporter through it (audit list of bypassers:
  map-sheet PDF, measurements, space report, integrity, point clouds). Add a
  UI affordance (Export exploratory‚Ä¶ / blocked) ‚Äî the gate currently never
  reaches a button.

### Tier 3 ‚Äî correctness & hardening
- **PR7 Result-type unions** (¬ß11). `ValidationResult`, `RegistrationResult`
  (with `overlapFraction`/`inlierFraction` ‚Äî note: the ICP robustness attempt
  was reverted because a centroid warm-start over outliers made trimming
  collapse; redo it with a robust/iterated warm start under this PR),
  `SurfacePrediction` (measured/interpolated/unsupported/outside-domain).
- **PR8 Validation corrections** (¬ß13). Two real leaks remain: (a) reclassify
  ground **inside each training fold** so RMSE is end-to-end, not
  interpolation-of-a-fixed-mask; (b) split calibration from the validation set
  it is scored against; (c) report signed **bias** and **NMAD** (only abs/sq
  are accumulated today), and promote the blocked RMSE into the headline.
- **PR9 Product-specific cross-checks** (¬ß14). The generic `crossCheck` is
  hardened but still generic and **dormant**; add raster/classification/contour/
  volume-specific comparators and wire the harness.

### Tier 4 ‚Äî platform
- **PR11 Coverage contract** (¬ß18), **PR12 Workers/queues** (¬ß19, wire the
  dormant `pipelineLimits`/`gpuUploadQueue` or delete them), **PR10 Export
  consolidation** (¬ß15), **PR13 Bundle/deps** (¬ß20/22 ‚Äî `main.ts`/`Viewer.ts`
  decomposition to reclaim headroom), **PR14 Test infra** (¬ß27, property +
  mutation + worker-stability), **PR15 Docs + release gates** (¬ß28/31).
