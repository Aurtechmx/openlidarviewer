Binary share/icon assets — maintainer TODO
==========================================

index.html now references three binary assets that cannot be committed as
text and should be generated by the maintainer, then dropped into this
public/ directory (Vite copies public/ verbatim to the dist root):

1. og-card.jpg          — the Open Graph / Twitter share image
                          (1200x630 recommended). Suggestion: reuse the
                          existing promo screenshot at
                          docs/screenshots/openlidarviewer-promo.jpg
                          (crop/resize to 1200x630, save as og-card.jpg).

2. favicon.ico          — 32x32 (optionally multi-size 16/32/48) legacy
                          favicon. Generate from favicon.svg, e.g.:
                            rsvg-convert -w 32 -h 32 favicon.svg > favicon-32.png
                            convert favicon-32.png favicon.ico
                          or use any favicon generator with favicon.svg
                          as the source.

3. apple-touch-icon.png — 180x180 PNG, no transparency (iOS composites
                          it on white otherwise). Render favicon.svg at
                          180x180 over the #0a0e1a background.

Until these exist, browsers fall back gracefully: favicon.svg covers
every modern browser, and link previews simply omit the image.
