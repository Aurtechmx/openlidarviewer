# AI assistance

Generative AI tools were used during development of OpenLiDARViewer, including
v0.5.9 through v0.6.0-alpha.3: drafting and revising code and tests, reviewing changes for edge cases
and defects, editing documentation, and preparing release checks. Their output
was treated as suggestions, not decisions — reviewed against the existing
architecture, changed where needed, and validated through source inspection, the
automated test suite, production and obfuscated builds, browser tests, dependency
audits, and packaging checks.

The alpha.3 slope cross-implementation (E4) result was produced by the same
separation of roles. Development and documentation were human-directed with
AI-assisted code and doc review. The independent reference — the GDAL 3.13.1
slope raster — was generated by GDAL itself, an external tool AI did not write,
run inside, or influence. The numerical comparison is an automated test
(`tests/slopeCrossCheck.test.ts`) that reads the committed reference and asserts
agreement within the preregistered tolerance. AI did not generate, validate, or
vouch for the GDAL output; it is the independent artifact the claim rests on.

No benchmark, test result, citation, compatibility statement, or scientific
conclusion is presented as verified unless it is backed by the repository, an
executed test, a documented measurement, or a reviewed primary source. Generated
values were never accepted as evidence. What is and is not validated is recorded
in [VALIDATION_REPORT_v0.6.0-alpha.3.md](VALIDATION_REPORT_v0.6.0-alpha.3.md)
(with the terrain/measurement claims inherited from
[VALIDATION_REPORT_v0.5.9.md](VALIDATION_REPORT_v0.5.9.md)).

Scope, architecture, scientific boundaries, security requirements, interpretation
of results, and the decision to release remained the responsibility of the
maintainer, A. Urias. AI tools are not authors or contributors and are not
credited as such.
